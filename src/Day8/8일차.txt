멤버변수의 초기화
 -명시적 초기화 int x=0;
	       static int y=0;
 -생성자
 -초기화 블럭   {
	       	x=0;
	       }
	static {
		y=0;
	        }
 -초기화 순서
  기본값 -> 명시적 초기화 -> 클래스 초기화 블럭
  기본값 -> 명시적 초기화 -> 객체 초기화 블럭 -> 생성자
상속 : 부모 클래스한테 멤버변수나 멤버 메소드를 물려받는 것
        클래스를 선언할 때 받는 것
        부모 클래스의 public, protected로 되어있는 멤버변수, 멤버메소드를 사용할 수있다.
        생성자와 초기화블럭은 상속되지 않지만 부모의 생성자를 호출할 수 있다.
        부모클래스는 1개만 가능
        A는 B이다 => 상속, A:자식 B:부모
        A는 B를 가지고 있다. => 포함, A:클래스 B:멤버변수
class 클래스명 extends 부모클래스명 {
}
        스마트폰,카메라 => 포함, 클래스 : 스마트폰 멤버변수 : 카메라
상속과 포함의 차이
 상속은 멤버변수를 사용하기 위해 초기화를 하지 않아도 된다.
 포함에서 멤버변수는 new를 이용하여 객체를 생성후 해당객체의 멤버 변수를 사용할 수 있다. 
 포함은 클래스A의 멤버 변수로 클래스 B의 객체가 오는 것

public : 해당클래스 + 패키지 + 자식클래스 + 일반클래스
protected : 해당클래스 + 패키지 + 자식클래스
default : 해당클래스 +패키지
private : 해당클래스

this()와 super()는 생성자 코드에서만 사용가능하다.
this는 내 객체정보, this()는 내 생성자, super는 부모 객체정보, super()는 부모 생성자
